How the Simulation Works

The simulation engine is built on a loop that advances time in hourly increments. For a 60-day simulation, this loop runs 1,440 times (60 days * 24 hours). In each hourly step, the following sequence of events occurs:

1. Waste Accumulation

  • The engine checks if the current hour is within working hours (e.g., 08:00-16:00).
  • If it is, each station's current_waste_kg is increased by its hourly accumulation rate. Outside of working hours, waste levels remain constant. This strictly applies the "8-Hour World" logic.

2. Demand Trigger & Capacity Control

  • After waste is added, the engine checks each station.
  • If a station's current waste exceeds the defined trigger percentage (e.g., 85%), a "collection request" is created and added to the collection_queue, if one doesn't already exist for that station.
  • Simultaneously, if a station's waste exceeds 100% capacity while it's already waiting for collection, a "Service Failure" (overflow) is logged.

3. Truck Assignment Logic (The Critical Step)

  • The engine checks for pending requests in the collection queue.
  • It looks for an available truck (where is_busy = false).
  • It takes the oldest request from the queue and calculates a "trip plan":
    • Time from Garage to Station (read from the pre-calculated time matrix).
    • Loading time at the station (from parameters).
    • Time from Station back to Garage.
    • Unloading time at the garage.
  • It sums these to get the total trip duration.
  • "Is there enough time?" Check: The engine verifies if this total trip duration is less than the remaining time until the end of the current workday. If the trip cannot be completed before the workday ends, the assignment is not made.
  • If an available truck is found and the trip can be completed within work hours, the assignment is made:
    • The truck is marked as "busy," and its next availability time (busy_until_hour) is calculated.
    • The station's waste level is reset to zero.
    • Data such as the amount of waste collected is logged.
    • The request is removed from the collection queue.

How the New Outputs are Calculated

  • Potential Additional Tonnage: This metric quantifies the opportunity cost of idle truck time against a configurable target.
    1. Calculate Target Busy Time: `Target Busy Hours` = `Total Available Work Hours` * (`Target Utilization %` / 100).
    2. Find Usable Idle Time: `Usable Idle Hours` = `Target Busy Hours` - `Current Busy Hours`. If this is negative (i.e., current utilization is already above target), the potential gain is zero.
    3. Calculate Average Trip Efficiency: The simulation finds the `Average Tonnage per Trip` and `Average Duration per Trip` from all completed collections.
    4. Calculate Potential: `Potential Extra Trips` = `Usable Idle Hours` / `Average Duration per Trip`. The final result is `Potential Extra Trips` * `Average Tonnage per Trip`, representing the additional tonnage that could be collected.
  • Average Station Wait Time: This chart measures the total time from request to truck arrival.
    1. Log Timestamps: When a station creates a request, the simulation logs the `request_hour`. When a truck is assigned, the `assignment_hour` is noted.
    2. Calculate Arrival Time: The truck's physical arrival time at the station is calculated: `arrival_hour` = `assignment_hour` + `travel_time_to_station`.
    3. Calculate Total Wait Time: For each collection, `Total Wait Time` = `arrival_hour` - `request_hour`.
    4. Average Results: After the simulation, the engine averages these total wait times for each station.
  • Hourly Truck Activity: This graph shows the operational rhythm of the workday.
    1. Track Activity: The simulation uses a 24-element array (one for each hour). For every hour of the simulation, it increments the counter for the current hour if a truck is busy.
    2. Normalize Data: At the end, each hourly total is divided by the number of simulation days to get the `daily average of active trucks` for each hour. This visualizes peak and off-peak operational times.

Key Assumptions for this Model

  • Sufficient Truck Capacity: The model assumes that any single truck has enough capacity to collect all the accumulated waste from any single station in one trip. The truck's capacity is not a constraint in this "As-Is" direct shipment model.
  • Direct Shipment Only: The simulation strictly follows a "one trip, one station" rule. A truck always returns to the garage after visiting a single station and does not perform multi-stop routes.
  • Constant Waste Rate: The simulation uses a fixed, average daily waste accumulation rate for each station. It does not account for daily or weekly fluctuations (e.g., more waste on weekends).
  • No Seasonality: The model does not consider seasonal effects that might increase or decrease waste generation over longer periods.
  • Perfect Information: The system instantly knows when a station reaches its fill level trigger. There are no communication delays between the station and the central depot.
